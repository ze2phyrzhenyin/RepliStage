# 项目概览

## 1. 产品定位

RepliStage 是一个通用的戏剧排练与导演工作台。

它的核心不是“页面写死一个剧本”，而是：

- 用统一 JSON 模型描述整部剧目
- 用同一套事件模型驱动排练和编辑
- 用浏览器本地存储承载当前工作剧本

当前仓库内置两部示例剧本：

- `Cendrillon`
- `秦琼卖马`

但系统本身并不依赖这两部剧本存在，导入符合规范的 JSON 后，同一套流程依然成立。

## 2. 当前入口流程

首页现在只做模式选择：

- `开始排练`
- `导演模式`

### 排练流程

1. 进入 `/rehearsal/setup`
2. 选择示例剧本或导入 JSON
3. 选择第几节
4. 进入 `/select-role`
5. 选择角色或旁观者
6. 进入 `/rehearsal`

### 导演流程

1. 进入 `/director/setup`
2. 选择示例剧本或导入 JSON
3. 选择第几节
4. 进入 `/director`

## 3. 已实现能力

- 多剧本示例库
- 多场次结构
- JSON 导入 / 导出
- 排练模式
- 导演模式
- 排练页轻量编排
- 舞台角色走位、路径走位、台词、动作、黑幕、退场
- 门和椅子等道具布景编辑
- 角色造型系统
- 中法双语界面
- 本地存储版本迁移
- 逻辑测试与构建验证

## 4. 当前边界

当前不是平台化系统，仍是单机前端工具。

未实现：

- 登录
- 权限
- 云端剧本存储
- 跨设备同步
- 版本历史 / 回滚
- 多人协作

## 5. 为什么当前结构成立

RepliStage 能保持可维护，关键在于三点：

- 同一份 `Play` 数据既能播放，也能编辑
- 播放逻辑集中在 `src/lib/player.ts`
- 当前工作剧本集中由 `PlayContext` 管理

所以：

- setup 流程只负责“当前用哪份剧本、哪一节”
- 排练页只负责“如何练”
- 导演页只负责“如何改”

## 6. 当前示例剧本说明

### `Cendrillon`

- 法语示例剧本
- 当前已扩展为多场次
- 适合验证多角色、多场次、成长线剧情和宫殿 / 玻璃房等不同场景

### `秦琼卖马`

- 中文示例剧本
- 当前包含两场
- 适合验证中文文本、室内戏、道具交互和人物关系推进

## 7. 推荐后续方向

- 增加更多剧本示例
- 为 JSON 导入增加差异预览
- 增加更高层的交互测试
- 加入版本历史和回滚
- 如果要上云，再抽离剧本仓储层
