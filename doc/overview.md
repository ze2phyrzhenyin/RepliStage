# 项目概览

## 1. 项目定位

StageCue 是一个为固定剧本排练场景打造的前端工具。它把“剧本事件”转成“舞台状态”，帮助演员在视觉化舞台中完成：

- 背词
- 对手戏节奏练习
- 走位记忆
- 场次推进与节拍感知

它不是通用剧本管理平台，也不是完整剧院生产系统。当前设计目标非常明确：

- 先把单剧本排练体验做到可用
- 用较低成本支持导演在本地直接编辑剧本
- 通过强类型事件模型保证渲染和编辑可以复用同一份数据

## 2. 当前功能边界

### 已实现

- 首页、角色选择、排练页、导演模式
- 多场次剧目结构
- 线性事件驱动回放
- 台词隐藏与逐句揭示
- 舞台人物、道具、黑幕、提示气泡
- 本地剧本编辑、JSON 导入导出与示例剧本恢复
- 角色造型自定义

### 未实现

- 多用户账号
- 远程协作
- 云端剧本存储
- 审批、版本历史、回滚界面
- 生产级权限控制
- 自动化测试体系

## 3. 页面与角色

### 页面

- `/`
  品牌首页，提供排练入口、场次切换与导演模式入口
- `/select-role`
  选择演员角色或以旁观者身份进入
- `/rehearsal`
  主排练工作区，包含演出、台本、编排三种模式
- `/director`
  导演模式，多场次编辑入口
- `/about`
  极简玩法说明

### 使用角色

- 演员
  进入排练页，以自己角色身份背词
- 旁观者
  可看到完整台词，适合导演跟本或教学
- 导演/开发者
  使用排练页编排模式或导演模式编辑剧本

## 4. 运行模式

### 普通运行

使用 `npm run dev` 启动，所有排练功能可用。

### 导演模式

`/director` 使用浏览器中的当前剧本数据工作，支持导入 JSON、导出 JSON 和恢复示例剧本。

### 排练页编排模式

排练页中的“编排”模式不写文件，只写浏览器本地存储，并且按场次隔离。适合临时调整。

## 5. 核心数据源

- 运行时默认剧本：`lib/default-script.json`
- 类型约束：`types/script.ts`
- 播放状态推导：`lib/player.ts`

播放器并不是通过请求远程接口获取状态，而是直接从本地剧本数据派生当前舞台状态。

## 6. 为什么这个项目容易维护

原因主要有三点：

- 剧本结构集中在一份强类型 JSON/TypeScript 模型中
- 播放逻辑集中在 `deriveStageState()`，可预测、可读
- 排练端和导演端共用相同事件模型，减少了双份业务规则

## 7. 建议的后续演进

- 抽离剧本仓储层，支持文件与远程存储双模式
- 给事件模型增加更严格的业务校验
- 增加自动化测试，覆盖播放器派生逻辑
- 把导演模式拆成更稳定的编辑器模块
- 引入版本化与差异比对，避免 JSON 导入后被误覆盖
